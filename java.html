1. Introducción

Java es un lenguaje de programación orientado a objetos (POO), el cual es muy popular y nos permite hacer desde aplicaciones de escritorio básicas, hasta videojuegos, webs o aplicaciones móviles. A lo largo de esta lección aprenderás la sintaxis para escribir programas en java y los conceptos asociados a las clases y la programación orientada a objetos.

2.1 Comentarios

Los comentarios en Java son iguales que los de C. Se usa la secuencia /* para abrir el comentario y la secuencia */ para cerrarlo. También se permite usando la doble barra // para comentarios de una linea.

3.Variables y tipos de datos

Las variables Java son un espacio en memoria que almacena un valor que necesitamos y que manipulamos. La sintaxis es la siguiente: 

        [modificador de acceso] tipo_variable identificador;

El tipo de variable puede ser una clase o un tipo primitivo que explicaremos a continuación. El identificador es el nombre (único) que se le da la variable. Hay que tener en cuenta que se distinguen entre mayusculas y minúsculas y hay una serie de normas a seguir. No se puede comenzar el nombre de una variable por un número o un signo de puntuación, y normalmente (no obligatorio) se suele empezar con una letra.
El modificador de acceso es explicado más adelante.

5. Conceptos de POO

La programación orientada a objetos (POO) funciona mediante la creación de clases y objetos que almacenan tanto datos como métodos. La POO tiene ciertas ventajas sobre la programación por procedimientos.

  • La POO es más rapida y facil de ejecutar
  • Proporciona una estructura más clara para los programas
  • Ayuda a seguir el principio DRY (Don't Repeat Yourself) lo que hace que el código sea mucho más reutilizable, facil de depurar y mantener
5.1 Clases/Objetos

Clase Las clases representan los prototipos de los objetos que tenemos en el mundo real. Es decir, es una generalización de un conjunto de objetos que guardan relación entre ellos, compartiendo atributos y funcionalidades comunes. A su vez los objetos serán instancias de una determinada clase.
Un ejemplo de de clase sería la siguiente: 

                public class Persona {
                  private string nombre;
                  private string apellidos;
                  public int edad;
                  private string email;

                  public void andar()
                  public void comer()
                  public void dormir()
                }

Esta clase tendrá una serie de atributos, como son propiedades asignables a una persona como su nombre,apellidos, edad o su correo eléctronico. Además tendrá una serie de métodos que son funcionalidades que una persona puede hacer como andar, comer o dormir.
En Java, se usa el operador . para acceder a los campos de un objeto y el operador new para instanciarlos.

                    
                Persona miPersona = new Persona();
                miPersona.andar();
                miPersona.edad = 28;
5.2 Modificadores de acceso

Los modificadores de acceso son palabras claves que se usan para definir el nivel de acceso para las clases, atributos y métodos. Concretamente para las clases tenemos lo siguiente:

Modificador Descripción
public El código es accesible por todas las clases
private El código solo es accesible desde dentro de la clase
default/package El código es accesible por todas las clases del mismo paquete
protected Como package pero además accesible desde las subclases

5.3 Encapsulación de datos

Las interacciones con los objetos se hacen mediante los métodos. Es decir, si queremos conocer información del estado del objeto deberemos de llamar a uno de sus métodos y no directamente a las propiedades. Esta encapsulación también nos permitiría actualizar los atributos del objeto. Esto nos aporta los siguientes beneficios:

  • Encapsular datos : ocultamos la implementación de propiedades del objeto ya que accederemos a través de los métodos del objeto, ocultando posible información sensible.
  • Reutilización de Código : podemos tener diferentes instancias de un objeto de tal manera que esas diferentes instancias están compartiendo el mismo código.

Para implementar la encapsulación, lo recomendable es que los atributos estén declarados como privados, y se accedan a ellos mediante funciones publicas denominadas set y get. Siguiendo el ejemplo de persona, anteriormente le asignabamos a nuestro objeto miPersona una edad de 28 años porque el atributo edad era público, a diferencia de los otros.
Sin embargo, si ahora lo hacemos privado, nos dará un error al intentar asignarle un valor a una variable a la cual no tenemos acceso. Para ello crearemos 2 funciones llamadas getEdad y setEdad para obtener el valor y para actualizarlo correctamente. 

                public class Persona {
  
                  private int edad;

                  public void setEdad ( int edad ) {
                    this.edad = edad;
                  }
                  public int getEdad (){
                    return this.edad;
                  }

                 public static void main(String[] args) {
                 Persona miPersona = new Persona();
                 miPersona.setEdad(28);
                    }

La palabra reservada this, es usada para referirnos al objeto actual que llamó a dicha función. Cuando se llama a setEdad, this se referirá al atributo edad del objeto miPersona que hemos creado y que llamó a dicho método.

5.4 Constructores

Los constructores son métodos especiales usados para la inicialización de los objetos. Al crearlos, en vez de llamar a los métodos sets, se usan los constructores para indicar que valores tendrá el objeto.
Siguiendo los ejemplos de la clase persona, un constructor sería como el siguiente: 

                 public class Persona {
                    public Persona(string nombre, string apellidos, int edad, string correo){
                        this.nombre = nombre;
                        this.apellidos = apellidos;
                        this.edad = edad;
                        this.email = correo;
                        }
                }

El nombre del constructor debe ser el mismo que el de la clase. Este será llamado automáticamente al crear el objeto

                 public static void main(String[] args) {
                 Persona miPersona = new Persona("Miguel","López García", 30, "miguelopezgarcia@gmail.com");
                 }
5.5 Paquete

Un paquete es una forma de organizar elementos de software mediante un espacio de nombres. Así podremos afrontar desarrollos grandes de software facilitando la forma de encontrar o referirnos a un elemento. Podríamos entender el sistema de paquetes como si fuese un sistema de carpetas. De tal manera que colocaremos cada una de las clases (o ficheros) en un paquete (o directorio). Los paquetes se definen mediante el modficiador package seguido del nombre del paquete. El paquete lo definiremos en la primera línea de cada una de las clases.

5.6 Herencia

La herencia es una forma de estructurar el software. Mediante la herencia podemos indicar que una clase ( subclase o clase hija ) hereda de otra ( superclase o clase padre) . Es decir la clase extiende las capacidades (propiedades y métodos) que tenga y añade nuevas propiedades y acciones. Digamos que las nuevas clases especializan más aquellas clases de las que heredan al añadir nueva funcionalidad. Aunque también pueden reescribir el funcionamiento de dichos elementos. En Java se usa la palabra extends para implementar la herencia.
Volviendo al ejemplo de la clase persona anterior, podemos tener ahora dos clases que heredan de ella: alumno y profesor y cada uno tendrá unas características concretas. 

            public class Alumno extends Persona {
            
            private int notaMedia;

            public void presentarseExamen();


            }   

            public class Profesor extends Persona {

            private int añosExperiencia;
            
            public void publicarNotas();
            }

La subclase, heredará todos los métodos y atributos de la clase padre, e implementará otros nuevos más específicos. De esta forma, tanto alumno como profesor, serán personas, y tendrán metodos como andar,comer o dormir y atributos como el nombre y la edad, pero se diferenciarán ya que el alumno por ejemplo se presentará a exámenes y el profesor publicará las notas de los alumnos.
Para acceder a los miembros de la superclase se usa la palabra reservada super. Es similar a this, pero referido a la clase padre. 

            public class Alumno extends Persona {
            
            public Alumno ( int notaMedia, string nombre, string apellidos, int edad, string correo) {
                super(nombre,apellidos,edad,correo);
                this.notaMedia = notaMedia;
                }
5.7 Polimorfismo

En programación orientada a objetos el polimorfismo es la capacidad que tiene un objeto de responder de maneras diferentes ante mismos mensajes envíados a objetos En Java es implementado mediante la sobrecarga de métodos, que nos permite tener un mismo método con diferentes firmas. En función de los parámetros y los tipos de los mismos, el compilador decide a qué método se está llamando. De ésta manera podemos realizar diferentes tareas según los parámetros que le pasemos a la función.
Siguiendo con los ejemplos anteriores de alumno, profesor y persona, podríamos crear un método que nos imprima la información de la persona con diferentes funciones según si es alumno o profesor.

5.8 Interfaz

Un interfaz es una forma de establecer un contrato entre dos elementos. Un interfaz indica qué acciones son las que una determinada clase nos va a ofrecer cuando vayamos a utilizarla. Cuando implementemos un interfaz (cuando lo usemos) deberemos de implementar todas las acciones (métodos) que este contenga.

            public class Persona 
             {

              public void imprimeInfo()
               System.out.println("Mi nombre es : " + this.nombre);
             }

            public class Alumno extends Persona 
              {
              public string destinoErasmus

              public void imprimeInfo()
                System.out.println("Mi nombre es : " + this.nombre + " y mi nota media es : " + this.notaMedia);

              public void imprimeInfo(boolean erasmus)
                {
                System.out.println("El alumno : " + this.nombre + "el año que viene se irá de Erasmus a " + this.destinoErasmus);
                }
              }
            public static void main(String[] args) {
                Alumno miAlumno = new Alumno(8, "Manuel", "Jimenez Garcia", 20,manueljg@gmail.com, Londres);
                miAlumno.imprimeInfo(); // Imprime  Mi nombre es Manuel y mi nota media es 8
                miAlumno.imprimeInfo(true); // Imprime El alumno Manuel el año que viene se irá de Erasmus a Londres
            }

Mediante la misma llamada ( con diferentes parámetros ) tenemos dos respuestas diferentes. De la misma manera, podríamos tener otro método imprimeInfo en profesor con una salida diferente. De esta forma, podemos hacer nuestro código mucho más reutilizable y facil de mantener.
